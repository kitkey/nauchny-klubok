# Научный клубок — граф знаний R&D + интеллектуальный поиск

Превращает корпус документов (PDF, RU/EN) в **граф знаний** и отвечает на вопросы исследователя как
литературный обзор — со ссылками на точные источники, числами из данных, подсветкой противоречий и
пробелов. Домен: горно-металлургические R&D.

- **Построение графа** — по подходу **EDC (Extract–Define–Canonicalize)**: LLM извлекает n-арные факты,
  система канонизирует сущности и разрешает конфликты. Факты типизированы (материал / процесс /
  свойство / оборудование / площадка / человек / условие + `Quantity`).
- **Поиск** — векторный по графу с декомпозицией мультивопросов + **PageRank (Neo4j GDS)** (в духе
  **HippoRAG**); ответ синтезируется из фактов, числа берутся из графа, не из модели.
- **Верификация** — **ленивая, в момент ответа**: факт, попавший в ответ, сверяется с источником и его
  статус кэшируется в граф (повторно не проверяется).
- **Хранение** — **Neo4j** (граф + нативный vector-index + GDS) + **MongoDB** (оригиналы документов).
- **Веб** — основной UI (спросить / документы / трейс / дашборд) **+ интерактивный граф-эксплорер**
  (`/explore`) с фильтрами, географией, институциональной памятью и поиском внешних источников.
- **LLM** — провайдер-агностично: **OpenRouter** (DeepSeek V4 Flash) или **Yandex AI Studio** (российское
  облако, ~1.3–1.6× быстрее с reasoning-off).

## Быстрый старт

Нужны: Python ≥ 3.12, Docker, ключ LLM (OpenRouter или Yandex).

```bash
cp .env.example .env          # впиши OPENROUTER_API_KEY (или YANDEX_API_KEY + YANDEX_FOLDER_ID)
make up                       # поднимет Neo4j+GDS, Mongo и приложение на :8000 (полный стек одной командой)
```

Без make:
```bash
docker compose -f deploy/docker-compose.yml up -d   # neo4j(+GDS) + mongo + app
# открыть http://localhost:8000
```

Открой **http://localhost:8000** → вкладка «Документы» загрузи PDF → «Спросить» задай вопрос,
или **http://localhost:8000/explore** — интерактивный граф.

## Прогон всего пайплайна на документах

### Вариант 1 — CLI по папке (пакетно)

```bash
python bulk_ingest.py --dir ./docs --graph "Мой корпус"
# на Yandex, первые 50, русский вывод:
python bulk_ingest.py --dir ./docs --graph "Корпус" --provider yandex --limit 50 --lang ru
```

Пайплайн **S0→persist** на каждый документ:

| Стадия | Что делает |
|---|---|
| **S0 ingest** | PDF → текст (PyMuPDF, OCR-фолбэк), разметка секций/таблиц |
| **S1 chunk** | деление на смысловые фрагменты под токен-бюджет |
| **S2 roles** | риторические роли пассажей *(в bulk выключается)* |
| **S3 extract** | LLM извлекает **n-арные факты** с участниками и числами |
| **S4 link** | дедуп сущностей, канонизация, конфликты (`contested`), сшивка |
| **S5 verify** | грунт-чек факта *(в bulk выключается — верифицируем лениво при запросе, см. ниже)* |
| **persist** | Neo4j (узлы/рёбра) + Mongo (оригиналы) + vector-index (эмбеддинги) |

**вытесняет старую версию** (версионирование).

## Как работает поиск

1. Вопрос → LLM разбивает мультивопрос на **под-темы** (декомпозиция).
2. **Векторный поиск** по `Fact.statement` в Neo4j — семантика, кросс-документно. Адаптивный **порог
   релевантности** отсекает слабый хвост (шумовые документы).
3. **PageRank (GDS)** — персонализированный PPR от якорных сущностей по графу `Entity+Fact+Concept` +
   рёбра REL + **мост по одинаковому `key`** (кросс-док) → факты, связанные структурно.
4. **Диверсификация** (round-robin) + **число-предпочтение**: факты с `q_value` чередуются с
   качественными, чтобы числовые не вытеснялись из окна (`k = 16` по умолчанию).
5. Для топ-фактов — окрестность (участники) bounded-Cypher'ом (1 hop).
6. **Синтез**: числа из полей графа, ссылки на источники, консенсус/расхождения, пробелы и рекомендации.

**Кросс-документность** идёт через (а) семантику векторов и (б) PageRank + детерминированный мост по
`key`. LLM-концепт-слой дал бы более сильную связность (синонимы/аббревиатуры) — см. Roadmap.

## Верификация фактов (ленивая, при запросе)

Массовый S5-verify в bulk отключён — факты остаются `unverified`. Вместо этого верификация идёт
**лениво в момент ответа** (`cfg.verify_on_read`, по умолчанию вкл):

- проверяются только факты, реально попавшие в ответ, и только ещё не проверявшиеся;
- источник для сверки реконструируется из **units в Mongo** (осмысленные пассажи статьи);
- LLM-судья по 3 осям (confidence/clarity/relevance) + порог → `verified` либо остаётся `unverified`,
  но ставится флаг `v_checked` — **повторно факт не гоняется** (разовая цена, амортизируется);
- `verified` = «факт заземлён на свой источник», а не «истина в мире»; противоречия остаются `contested`
  и честно показываются, а не выбираются молча.

Код: `p2kg/verify.py` (`s5_verify`, `verify_on_read`), статус пишется в Neo4j (`graphstore.set_fact_verification`).

## Граф-эксплорер (`/explore`)

Отдельная полноэкранная страница (`p2kg/api/static/explore/`) — интерактивный граф на vis-network:

- **Канвас** — реальный **срез** графа (~сотни узлов: приоритет противоречивых/verified/числовых фактов
  + их сущности, публикации, эксперты). Полный граф (десятки тысяч узлов) в браузер не грузим —
  вопросы по нему идут через **чат** (см. ниже). Клик по узлу → факт, число, источник, достоверность.
- **Фильтры** — по типу сущности и **география РФ / мир**.
- **Аналитика** — институциональная память (кто держит экспертизу и риск её потери), пробелы,
  карта противоречий, heatmap «материал × условие».
- **Чат** — отвечает по **полному** графу через `/api/graphs/{gid}/ask` (без внешних LLM — суверенно).
- **🌐 Внешние источники** — поиск публикаций по теме, которых нет в базе (см. ниже).

Данные канваса генерируются из Neo4j (реальный срез) в формат фронта; гео и авторы/организации
проставляются обогащающим проходом (LLM по титулам статей).

## Внешние источники (article-discovery)

Эндпоинт `GET /api/graphs/{gid}/discover?topic=…`: LLM формирует английский научный запрос →
**Crossref** (реальные публикации, без ключа) → дедуп против названий корпуса → публикации, которых
**нет в базе**, со ссылками. Наружу уходит только строка-запрос, не данные корпуса; результаты —
кандидаты на добавление, требуют проверки экспертом. Без галлюцинированных ссылок.

## Веб-интерфейс

- **Спросить** — NL-вопрос → ответ-обзор с карточками, источниками, экспортом в Markdown.
- **Документы** — загрузка PDF, статус ингеста, версионирование.
- **Трейс** — интерактивный граф: из каких фактов/сущностей собран последний ответ.
- **Дашборд** — покрытие по полному графу: сущности/факты по типам, источники, спорные факты.
- **Граф знаний ↗** — переход в граф-эксплорер (`/explore`).

## Структура проекта

```
p2kg/
  ingest/      S0: PDF → Paper
  chunk.py     S1: чанкинг
  roles.py     S2: риторические роли
  extract/     S3: n-арное извлечение (EDC)
  link/        S4: дедуп/канон/конфликты/сшивка + concepts.py (концепт-слой)
  verify.py    S5: грунт-чек + ленивая верификация при запросе (verify_on_read)
  retrieve.py  ретрив: декомпозиция + vector + PageRank + число-предпочтение + синтез
  stores/      Neo4j (граф+vector+GDS+верификация) / Mongo (артефакты)
  llm/         клиент (OpenRouter/Yandex) + шаги + промпты
  api/         FastAPI: app.py, service.py (ask/discover), control.py, static/ (UI + explore/)
  schema.py    контракты данных (pydantic)
  pipeline.py  оркестрация S0..persist
bulk_ingest.py CLI прогона по папке
deploy/        docker-compose (Neo4j+GDS + Mongo + app) + Dockerfile
tests/         юнит-тесты (pytest)
docs/          IDEA / PIPELINE
```
