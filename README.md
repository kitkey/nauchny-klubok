# Научный клубок — граф знаний R&D + интеллектуальный поиск

Превращает корпус документов (PDF, RU/EN) в **граф знаний** и отвечает на вопросы исследователя как
литературный обзор — со ссылками на точные источники, числами из данных, подсветкой противоречий и
пробелов. Домен: горно-металлургические R&D.

- **Построение графа** — по подходу **EDC (Extract–Define–Canonicalize)**: LLM извлекает n-арные факты,
  система канонизирует сущности и разрешает конфликты. Факты типизированы (материал / процесс /
  свойство / оборудование / площадка / человек / условие + `Quantity`).
- **Поиск** — векторный по графу с декомпозицией мультивопросов (в духе **HippoRAG**), ответ синтезируется
  из фактов; числа берутся из графа, не из модели.
- **Хранение** — **Neo4j** (граф + нативный vector-index) + **MongoDB** (оригиналы документов).
- **LLM** — провайдер-агностично: **OpenRouter** (DeepSeek V4 Flash) или **Yandex AI Studio** (российское
  облако, ~1.3–1.6× быстрее с reasoning-off).

## Быстрый старт

Нужны: Python ≥ 3.12, Docker, ключ LLM (OpenRouter или Yandex).

```bash
cp .env.example .env          # впиши OPENROUTER_API_KEY (или YANDEX_API_KEY + YANDEX_FOLDER_ID)
make up                       # поднимет Neo4j+Mongo, установит пакет, запустит сервер :8000
```

Без make (Windows):
```bash
docker compose -f deploy/docker-compose.yml up -d
pip install -e ".[dev]"
uvicorn p2kg.api.app:app --port 8000
```

Открой **http://localhost:8000** → вкладка «Документы» загрузи PDF → вкладка «Спросить» задай вопрос.

## Прогон всего пайплайна на документах (главное)

### Вариант 1 — CLI по папке (пакетно)

```bash
# сложи PDF в ./docs, затем:
python bulk_ingest.py --dir ./docs --graph "Мой корпус"

# на Yandex, первые 50 документов, русский вывод:
python bulk_ingest.py --dir ./docs --graph "Корпус" --provider yandex --limit 50 --lang ru
```

Что произойдёт с каждым документом (пайплайн **S0→persist**):

| Стадия | Что делает |
|---|---|
| **S0 ingest** | PDF → текст (PyMuPDF, OCR-фолбэк), разметка секций/таблиц |
| **S1 chunk** | деление на смысловые фрагменты под токен-бюджет |
| **S2 roles** | риторические роли пассажей *(в bulk выключается)* |
| **S3 extract** | LLM извлекает **n-арные факты** с участниками и числами |
| **S4 link** | дедуп сущностей, канонизация, конфликты (`contested`), сшивка |
| **S5 verify** | грунт-чек факта по источнику *(в bulk выключается)* |
| **persist** | Neo4j (узлы/рёбра) + Mongo (оригиналы) + vector-index (эмбеддинги) |

По завершении граф виден в UI (левое меню). `--full` включает roles+verify (медленнее, полнее).

### Вариант 2 — через UI

Вкладка «Документы» → «Загрузить документ» (PDF). Ингест идёт фоном, статус там же. Загрузка **новой
версии того же файла вытесняет старую** (версионирование).

### Проверка результата

```bash
# спросить из терминала (или через UI «Спросить»):
curl -s -X POST localhost:8000/api/graphs/<GID>/ask \
  -H "Content-Type: application/json" -d '{"question":"Какие методы удаления SO2 описаны?"}'
```

## Как добавлять документы

- **UI:** вкладка «Документы» → загрузка PDF (по одному).
- **CLI:** `python bulk_ingest.py --dir <папка> --graph "<имя графа>"` — доливает в существующий/новый граф.
- **Инкрементально:** граф не пересобирается — новые документы доливаются, эмбеддинги считаются на лету.
- **Версионирование:** повторная загрузка того же `paper_ref` удаляет старые узлы документа и заливает новые.

## Как работает поиск

Два сигнала над графом, фьюзятся:
1. Вопрос → LLM разбивает мультивопрос на **под-темы**.
2. **Векторный поиск** по `Fact.statement` в Neo4j (эмбеддинги закэшированы при ингесте) — семантика, кросс-документно.
3. **PageRank (Neo4j GDS)** — персонализированный PPR от якорных сущностей (топ по под-темам) по графу
   `Entity+Fact+Concept` + рёбра REL + **мост по одинаковому `key`** (одноимённые сущности разных доков) →
   факты, связанные структурно. Требует плагин `graph-data-science` (включён в docker-compose); без него
   ретрив деградирует на чистый вектор.
4. **Диверсификация** (round-robin) — вектор и PPR представлены, один сигнал не забивает выдачу.
5. Для топ-фактов — окрестность (участники) bounded-Cypher'ом (1 hop).
6. **Синтез**: числа из полей графа, ссылки на источники, консенсус/расхождения, **пробелы и рекомендации**.

**Как связано (честно):** кросс-документность идёт через (а) **семантику векторов** (statement'ы разных
доков с близким смыслом — в одном поиске) и (б) **PageRank + детерминированный мост по `key`** (одноимённые
сущности из разных документов связываются в проекции GDS). Более сильную семантическую связность
(SO₂ ≈ диоксид серы) дал бы LLM-концепт-слой — см. ниже.

Ответ несёт: текст (лит-обзор), карточки-эвиденс (значение+участники+статус+источник), **точные
файлы-источники**, и **подграф трейса** (из каких фактов собран ответ — вкладка «Трейс»).

## Концепт-слой (кросс-документная связность)

Поверх конкретных сущностей задуман слой обобщённых понятий: `Entity —INSTANCE_OF→ Concept —SUBTYPE_OF→
Concept`. Он связывает одинаковые сущности из разных документов (в графе они — разные узлы) и даёт
отвечать по семействам, а не по одной марке. Код: `p2kg/link/concepts.py`, запуск —
`build_and_persist_concepts(deps)`.

**Статус (честно): концепт-слой на графах СЕЙЧАС НЕ построен** и в ретриве не используется. Кросс-док
достигается вектором + PageRank-мостом по одинаковому `key` (см. «Как работает поиск»). LLM-концепты
дали бы более сильную семантическую связность (синонимы/аббревиатуры), но на текущем масштабе
(~17k сущностей) это отдельный дорогой проход (≈$1, ~20 мин) с патчем батчинга эмбеддингов и
инкрементальным присоединением (загрузка концептов из Neo4j обратно) — вынесено в Roadmap.

## Веб-интерфейс

- **Спросить** — NL-вопрос → ответ с карточками, источниками, экспортом в Markdown.
- **Документы** — загрузка PDF, статус ингеста.
- **Трейс** — интерактивный граф: из каких фактов/сущностей собран последний ответ.
- **Дашборд** — покрытие: сущности/факты по типам, источники (зоны риска), спорные факты.

## LLM-провайдеры

| Провайдер | Модель | Особенность |
|---|---|---|
| OpenRouter (по умолч.) | `deepseek/deepseek-v4-flash` | reasoning гасится через `extra_body` |
| Yandex AI Studio | `gpt://<folder>/deepseek-v4-flash/latest` | российское облако, reasoning-off (`reasoning_effort:"none"`), быстрее |

Провайдер выбирается флагом `--provider` (CLI) или конфигом; прикладной код не меняется.

## Структура проекта

```
p2kg/
  ingest/      S0: PDF → Paper (адаптеры)
  chunk.py     S1: чанкинг
  roles.py     S2: риторические роли
  extract/     S3: n-арное извлечение (EDC)
  link/        S4: дедуп/канон/конфликты/сшивка + concepts.py (концепт-слой)
  verify.py    S5: грунт-чек
  retrieve.py  ретрив: декомпозиция + vector-search + синтез
  stores/      Neo4j (граф+vector) / Mongo (артефакты)
  llm/         клиент (OpenRouter/Yandex) + шаги + промпты
  api/         FastAPI: app.py, service.py, control.py, static/ (UI)
  schema.py    контракты данных (pydantic)
  pipeline.py  оркестрация S0..persist
bulk_ingest.py CLI прогона по папке
deploy/        docker-compose (Neo4j + Mongo)
tests/         юнит-тесты (pytest)
docs/          IDEA / PIPELINE
```

## Roadmap

- Концепт-обход в ретриве (раскрытие якорей + hop по INSTANCE_OF/SUBTYPE_OF) и инкрементальный
  концепт-слой (загрузка из Neo4j + триггер при ингесте).
- Числовые / географические / временны́е фильтры в UI (+ извлечение года из метаданных).
- Ручная правка графа экспертами, экспорт PDF/JSON-LD, дашборды-метрики, уведомления о новых публикациях.
- Коннекторы источников (elibrary / Springer / патенты) поверх модульного `ingest/`.
